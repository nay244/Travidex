# Travidex
Design system + spec live in `design/Travidex/`. Before building UI:
1. Read `design/Travidex/SKILL.md` (entry point) and `design/Travidex/readme.md` (design language).
2. Read `design/Travidex/Travidex-Screen-Flow-and-Design.md` for screens, interactions & data model.
3. Use tokens from `design/Travidex/styles.css`; match components in `design/Travidex/components/`.
Rules: light default + dark = `[data-theme="dark"]`; never branch components on theme.
Space Grotesk (UI) + Space Mono (data). Accents = fixed semantics. Found = full image, unfound = hollow.
